# 공실 상가 지도 웹뷰 PRD (Product Requirements Document)

## 1. 프로젝트 개요
모바일 앱에 내장될 웹뷰용 공실 상가 지도 서비스를 개발한다. 사용자가 지도에서 공실 상가 정보를 쉽게 찾아보고 상세 정보를 확인할 수 있는 웹 애플리케이션을 구축한다.

## 2. 기술 스택
- **프레임워크**: Next.js (React 기반)
- **지도 API**: 카카오맵 또는 네이버맵 (한국 서비스 최적화)
- **스타일링**: Tailwind CSS
- **상태관리**: React useState/Context API
- **타입스크립트**: TypeScript 사용
- **배포**: Vercel 또는 Next.js 스탠다드 배포

## 3. 핵심 기능

### 3.1 메인 지도 화면
- 전체 화면 지도 인터페이스
- 현재 위치 기반 지도 초기화
- 지도 확대/축소, 이동 기능
- 모바일 웹뷰 최적화된 UI/UX

### 3.2 공실 상가 마커 시스템
- 지도에 공실 상가 위치 마커 표시
- 마커 클러스터링 (많은 마커 최적화)
- 마커 아이콘 디자인 (공실 상가임을 명확히 표시)
- 마커 호버/포커스 효과

### 3.3 상세 정보 바텀시트
- 마커 클릭 시 하단에서 올라오는 바텀시트
- 드래그로 바텀시트 높이 조절 가능
- 상가 상세 정보 표시:
  - 상가명/건물명
  - 주소
  - 면적 (평수)
  - 월 임대료
  - 보증금
  - 연락처
  - 상가 이미지 (있는 경우)
  - 주변 시설 정보
- 바텀시트 닫기 기능

### 3.4 검색 및 필터 기능
- 지역별 검색
- 임대료 범위 필터
- 면적 범위 필터
- 업종별 필터 (음식점, 소매점, 사무실 등)

### 3.5 모바일 최적화
- 터치 제스처 최적화
- 반응형 디자인
- 빠른 로딩 속도
- 오프라인 기본 기능

## 4. 데이터 구조

### 4.1 공실 상가 데이터 모델
```typescript
interface VacantShop {
  id: string
  name: string
  address: string
  latitude: number
  longitude: number
  area: number // 평수
  monthlyRent: number // 월 임대료
  deposit: number // 보증금
  contactNumber: string
  images?: string[]
  description?: string
  shopType: 'restaurant' | 'retail' | 'office' | 'etc'
  availableFrom: Date
  features?: string[] // 주차가능, 1층, 코너상가 등
}
```

## 5. UI/UX 요구사항

### 5.1 디자인 가이드라인
- 모바일 퍼스트 디자인
- 최소 터치 타겟 크기: 44px
- 직관적인 네비게이션
- 빠른 정보 접근성
- 접근성 가이드라인 준수

### 5.2 성능 요구사항
- 초기 로딩 시간 < 3초
- 마커 렌더링 최적화
- 이미지 lazy loading
- API 응답 캐싱

## 6. 개발 단계

### Phase 1: 기본 설정 및 지도 구현
- Next.js 프로젝트 설정
- 지도 API 연동
- 기본 지도 화면 구현
- 마커 표시 기능

### Phase 2: 바텀시트 및 상세 정보
- 바텀시트 컴포넌트 개발
- 상세 정보 표시 UI
- 마커 클릭 이벤트 연동

### Phase 3: 데이터 관리 및 최적화
- 목업 데이터 생성
- API 연동 구조 설계
- 성능 최적화
- 모바일 최적화

### Phase 4: 추가 기능 및 완성도
- 검색 및 필터 기능
- 이미지 갤러리
- 에러 처리
- 최종 테스트 및 배포

## 7. 성공 지표
- 사용자 체감 로딩 속도
- 마커 정확도 및 성능
- 바텀시트 사용성
- 모바일 접근성 점수
- 전체적인 사용자 경험 만족도
